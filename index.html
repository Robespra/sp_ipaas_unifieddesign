Basic design principles

As a developer/business owner you may occasionally have to solve design problems. The following principles are relevant to the presentation of content and the user's experience :

  • Start with typography, make headings big
  • Use generous white space
  • Make the layout clear
  • Keep it simple
  • Modal for self-contained process : any process that requires a specific user task, decision, or acknowledgement
  • Provide immediate feedback on user's action
  • Always Keep an eye on performance and responsiveness

Form

We will be using UIkit form utilities, please keep in mind the following practices when designing forms

1. Placeholders

A placeholder is supposed to help users understand what's required from them. Disappearing placeholder text strains users' short-term memory. Please avoid placeholder text as a label and place it above the input, but below the label. This approach: * Can be translated, * Won’t look like pre-filled information, * Can be seen in low vision circumstances, * Won’t disappear when content is entered into the input, * Can include semantic markup and be styled via CSS.

2. Visually group related labels and fields

If the form asks about different topics, section it into separate groups of fields and name the groups accordingly

in the above form screenshot, organizational inputs and access inputs have been grouped in two groups

3. Present fields in a single column layout

Multiple columns interrupt the vertical momentum of moving down the form. Exceptions to this rule : short and/or logically related fields (e.i : City : State, zip code...)

4. Provide highly visible and specific error messages

5. Use Inline Validation.
